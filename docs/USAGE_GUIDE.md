## はじめに

Qui Browser VRは、Meta Quest、Pico、その他のWebXR対応VRデバイス向けに最適化された没入型Webブラウザです。

### 主な特徴

- ✅ **ハンドトラッキング** - コントローラー不要の操作
- ✅ **3D空間UI** - 快適な視野角に配置されたインターフェース
- ✅ **ジェスチャーコントロール** - 12種類の直感的なジェスチャー
- ✅ **音声コマンド** - 日本語音声操作
- ✅ **360°メディア** - 動画・画像の完全サポート
- ✅ **アクセシビリティ** - WCAG AAA準拠
- ✅ **90FPS** - 滑らかで快適な体験

## デバイス別セットアップ

### Meta Quest 2 / 3

1. **ブラウザからアクセス**
   - VRヘッドセットを装着
   - ブラウザアプリを起動
   - URLバーに以下を入力:
     ```
     https://your-domain.com/qui-browser-vr
     ```

2. **VRモード起動**
   - 画面右下の「VRモード」ボタンをクリック
   - または音声コマンド: **"VRモード"**

3. **ハンドトラッキング有効化**
   - Meta Quest設定 > 動作 > ハンドトラッキング > オン
   - ブラウザでハンドトラッキングを許可

4. **推奨設定**
   - リフレッシュレート: 90Hz (Quest 2) / 120Hz (Quest 3)
   - 解像度: 高
   - フレームレート優先: オン

### Pico 4 / Neo 3

1. **ブラウザアクセス**
   - Picoヘッドセットを装着
   - Picoブラウザを起動
   - URLにアクセス

2. **VRモード**
   - 「VRモード」ボタンをタップ
   - WebXR権限を許可

3. **ハンドトラッキング**
   - Pico設定 > 実験的機能 > ハンドトラッキング > オン
   - ブラウザで有効化

4. **最適化**
   - パフォーマンスモード: バランス
   - 解像度: 自動

### HTC Vive / その他

1. **WebXR対応ブラウザ**
   - Chrome、Edge、Firefoxの最新版
   - WebXRフラグを有効化（必要に応じて）

2. **アクセス**
   - PCでURLを開く
   - VRモードボタンをクリック
   - ヘッドセットを装着

3. **コントローラー設定**
   - SteamVR経由でコントローラーを接続
   - ブラウザでコントローラー入力を許可

## 基本操作

### コントローラー操作

#### Meta Quest / Pico コントローラー

**ナビゲーション**:
- **トリガー**: 選択/クリック
- **グリップ**: グラブ/スクロール
- **サムスティック**: ページスクロール
- **A/Xボタン**: メニュー表示
- **B/Yボタン**: 戻る

**ショートカット**:
- **トリガー長押し**: リンクを新しいタブで開く
- **グリップ+トリガー**: ズーム
- **サムスティック押し込み**: ページ更新

#### キーボードショートカット

VRキーボードまたは物理キーボード使用時:

```
Ctrl + L      URLバー表示
Ctrl + T      新しいタブ
Ctrl + W      タブを閉じる
Ctrl + Tab    次のタブ
Ctrl + ,      設定を開く
Ctrl + B      ブックマーク表示

Alt + ←       戻る
Alt + →       進む
F5            更新
Escape        メニュー/キーボードを閉じる

1-9           タブ1-9に切り替え
P             パフォーマンスモニター表示
```

### ハンドトラッキング操作

#### 基本ジェスチャー

**ピンチ（親指+人差し指）**:
- 用途: 選択、クリック
- やり方: 親指と人差し指をくっつける
- ![Pinch Gesture](https://example.com/pinch.gif)

**ポイント（人差し指）**:
- 用途: ホバー、ポインティング
- やり方: 人差し指を伸ばす

**グラブ（握る）**:
- 用途: ページをグラブしてスクロール
- やり方: 手を握る

**スワイプ**:
- 用途: ページ送り、タブ切り替え
- 方向:
  - **右スワイプ**: 次のページ/タブ
  - **左スワイプ**: 前のページ/タブ
  - **上スワイプ**: ページ上部へ
  - **下スワイプ**: ページ下部へ

**円を描く**:
- 用途: ズーム、回転
- やり方: 人差し指で空中に円を描く
  - 時計回り: ズームイン
  - 反時計回り: ズームアウト

**サムズアップ（親指を立てる）**:
- 用途: ブックマークに追加
- やり方: 親指を上に立てる

**ピースサイン**:
- 用途: スクリーンショット
- やり方: 人差し指と中指でVサイン

**握りこぶし**:
- 用途: メニューを閉じる、キャンセル
- やり方: 手を強く握る

#### 高度なジェスチャー

**ダブルピンチ**:
- 用途: 新しいタブを開く
- やり方: 素早く2回ピンチ

**ロングホールド**:
- 用途: コンテキストメニュー表示
- やり方: ピンチを1秒以上保持

**両手ピンチ**:
- 用途: ページズーム
- やり方: 両手でピンチして広げる/狭める

### 視線（Gaze）操作

1. **有効化**:
   ```
   設定 > 入力 > 視線コントロール > オン
   ```

2. **使い方**:
   - 要素を1秒間見つめる（デフォルト）
   - プログレスサークルが完了すると選択
   - 視線カーソルが表示されます

3. **設定調整**:
   ```
   滞留時間: 0.5秒 - 2.0秒
   カーソルサイズ: 小/中/大
   カーソル色: カスタマイズ可能
   ```

### 音声コマンド

#### 有効化

```
設定 > アクセシビリティ > 音声コントロール > オン
```

#### 日本語コマンド一覧

**ナビゲーション**:
- **"戻る"** - 前のページに戻る
- **"進む"** - 次のページに進む
- **"更新"** - ページを更新
- **"ホーム"** - ホームページに移動
- **"停止"** - ページ読み込みを停止

**タブ操作**:
- **"新しいタブ"** - 新しいタブを開く
- **"タブを閉じる"** - 現在のタブを閉じる
- **"次のタブ"** - 次のタブに切り替え
- **"前のタブ"** - 前のタブに切り替え

**ブックマーク**:
- **"ブックマーク"** - ブックマーク一覧を表示
- **"ブックマークに追加"** - 現在のページを追加
- **"ブックマーク削除"** - ブックマークを削除

**検索**:
- **"検索"** - 検索バーを表示
- **"〜を検索"** - 指定したキーワードで検索
  - 例: "猫を検索"

**設定**:
- **"設定"** - 設定画面を開く
- **"ヘルプ"** - ヘルプを表示
- **"履歴"** - 閲覧履歴を表示

**VRコントロール**:
- **"VRモード"** - VRモード切り替え
- **"パフォーマンス"** - パフォーマンスモニター表示
- **"環境変更"** - 環境カスタマイザー表示

## 高度な機能

### 3Dタブマネージャー

#### レイアウト切り替え

1. **メニューから**:
   ```
   メニュー > タブ > レイアウト選択
   ```

2. **レイアウトの種類**:

   **Arc（弧）** - デフォルト
   - タブが弧状に配置
   - 最大10タブを視野内に表示
   - 推奨: 3-7タブ

   **Stack（スタック）**
   - タブが縦に積み重なる
   - コンパクトな表示
   - 推奨: 5-10タブ

   **Grid（グリッド）**
   - タブが格子状に配置
   - 一覧性が高い
   - 推奨: 6-9タブ

   **Cylinder（円筒）**
   - タブが円筒状に配置
   - 360度回転可能
   - 推奨: 8-10タブ

3. **クイック切り替え**:
   - キー1-9: タブ番号で直接切り替え
   - Ctrl+Tab: 次のタブ
   - Ctrl+Shift+Tab: 前のタブ

4. **タブアニメーション**:
   - スムーズなカメラ移動
   - アクティブタブのパルス効果
   - ホバー時のハイライト

### 3Dブックマーク

#### 視覚化モード

1. **Grid（グリッド）**:
   ```javascript
   // 設定で選択、またはジェスチャー
   グリッドレイアウト: 3x3または4x4
   ```
   - カードが整然と配置
   - 簡単に探せる
   - 最大50個まで表示

2. **Sphere（球体）**:
   - ブックマークが球状に配置
   - 黄金角分布で配置
   - 360度回転で閲覧

3. **Wall（壁）**:
   - ブックマークが壁面に配置
   - Pinterest風レイアウト
   - カテゴリ別に整理

4. **Carousel（カルーセル）**:
   - 回転式表示
   - 左右にスワイプで回転
   - 大きなプレビュー

#### 操作方法

**追加**:
- サムズアップジェスチャー
- または音声: "ブックマークに追加"
- またはメニューから

**削除**:
- ブックマークカードの×ボタンをピンチ
- またはロングホールドで削除メニュー

**開く**:
- カードをピンチして選択
- または視線で1秒見つめる

**整理**:
```
メニュー > ブックマーク > 整理
- カテゴリ別
- 日付順
- 頻度順
- アルファベット順
```

### ジェスチャーマクロ

#### マクロの録画

1. **録画開始**:
   ```
   メニュー > ジェスチャー > 録画開始
   ```

2. **ジェスチャーを実行**:
   - 記録したいジェスチャーを順番に実行
   - 例: 右スワイプ → ピンチ → サムズアップ

3. **録画停止**:
   ```
   メニュー > ジェスチャー > 録画停止
   ```

4. **マクロに名前をつける**:
   ```
   例: "クイックナビ"
   ```

#### プリセットマクロ

**クイックナビゲーション**:
- ジェスチャー: 右スワイプ + ピンチ
- 動作: 次のページに移動して最初のリンクを開く

**タブコントロール**:
- ジェスチャー: 円を描く + 上スワイプ
- 動作: 新しいタブを開いてブックマークを表示

**ブックマーク保存**:
- ジェスチャー: サムズアップ + ピンチ
- 動作: ブックマークに追加してフィードバック表示

#### カスタムマクロ作成

```javascript
// 開発者向け
window.gestureMacro.createMacro('my-macro', [
  { type: 'swipeRight', threshold: 0.3 },
  { type: 'pinch', threshold: 0.8 },
  { type: 'thumbsUp' }
], [
  () => navigateForward(),
  () => selectFirstLink(),
  () => addBookmark()
]);
```

### UIレイアウト

#### プリセットレイアウト

1. **Default（デフォルト）**:
   - 正面、2.0m
   - 標準的な配置
   - ほとんどのタスクに最適

2. **Comfortable（快適）**:
   - やや下向き（-15°）
   - 1.8m距離
   - 長時間使用に最適

3. **Theater（シアター）**:
   - 広い視野角
   - 3.0m距離
   - メディア視聴に最適

4. **Floating（フローティング）**:
   - 上下左右に配置
   - 1.5m距離
   - マルチタスクに最適

#### レイアウト調整

```
メニュー > 表示 > UIレイアウト

位置:
  - 水平角度: -45° to 45°
  - 垂直角度: -30° to 30°
  - 距離: 0.5m - 5.0m

スケール:
  - UIサイズ: 0.5x - 2.0x
  - 不透明度: 50% - 100%
  - 回転: 0° - 360°

アンカーモード:
  - World: 固定位置
  - Smooth-Follow: スムーズに追従
  - Lazy-Follow: 閾値後に追従（推奨）
```

### 360°メディア

#### 360°動画再生

1. **動画を開く**:
   ```
   URLバーに360°動画のURLを入力
   または
   ファイルピッカーから選択
   ```

2. **再生モード選択**:
   - **360°**: 全球体投影
   - **180°**: 半球投影
   - **Flat**: 通常の平面動画

3. **コントロール**:
   ```
   Space: 再生/一時停止
   ← →: 10秒戻る/進む
   ↑ ↓: 音量調整
   M: ミュート
   F: フルスクリーン

   ジェスチャー:
   - ピンチ: 再生/一時停止
   - 左右スワイプ: シーク
   - 上下スワイプ: 音量
   ```

4. **空間オーディオ**:
   - 自動的に3D空間オーディオを適用
   - 頭の向きに応じて音源位置が変化

#### 360°画像表示

1. **画像を開く**:
   - 360°画像（equirectangular形式）をサポート
   - 最大8K解像度

2. **ナビゲーション**:
   - 頭を動かして周囲を見渡す
   - グラブジェスチャーで回転
   - ピンチアウトでズーム

3. **プログレッシブロード**:
   - 低解像度プレビュー → 高解像度
   - スムーズな読み込み体験

### パフォーマンスプロファイラー

#### 詳細分析

```
メニュー > ツール > パフォーマンスプロファイラー
```

**測定項目**:
- CPU使用率
- GPU使用率・メモリ
- メモリ使用量（ヒープ）
- ネットワーク（帯域幅、レイテンシ）
- レンダリング（描画呼び出し、三角形数、テクスチャ）

**ボトルネック検出**:
- CPU: 処理が重い
- GPU: 描画が重い
- Memory: メモリ不足
- Network: 通信が遅い

**最適化推奨**:
- プロファイラーが自動的に問題を検出
- 具体的な改善策を提示
- ワンクリックで最適化適用

#### データエクスポート

```
プロファイラー > エクスポート
- JSON形式
- CSV形式
- テキストレポート
```

## トラブルシューティング

### よくある問題

#### Q1: VRモードが起動しない

**原因**:
- WebXRサポートがない
- HTTPSでない
- ブラウザが古い

**解決策**:
```
1. WebXRサポート確認:
   chrome://flags で "WebXR" を検索 → 有効化

2. HTTPSでアクセス:
   http:// → https://

3. ブラウザ更新:
   最新版に更新
```

#### Q2: ハンドトラッキングが動作しない

**原因**:
- ハンドトラッキング無効
- 照明が暗い
- 手が範囲外

**解決策**:
```
1. デバイス設定でハンドトラッキング有効化
2. 十分な照明を確保
3. カメラの視野内に手を配置
4. ブラウザでハンドトラッキング権限を許可
```

#### Q3: パフォーマンスが低い（<72 FPS）

**原因**:
- 重いWebページ
- 設定が高すぎる
- バックグラウンドアプリ

**解決策**:
```
1. パフォーマンスプロファイラーで原因特定:
   P キー → ボトルネック確認

2. 設定を下げる:
   - 解像度スケール: 100% → 80%
   - アンチエイリアス: オフ
   - パーティクル: 無効

3. バックグラウンドアプリ終了

4. 自動最適化有効化:
   設定 > パフォーマンス > 自動最適化: オン
```

#### Q4: テキストがぼやける

**原因**:
- 解像度が低い
- フォントサイズが小さい
- 距離が遠い

**解決策**:
```
1. テキスト拡大:
   設定 > アクセシビリティ > テキスト拡大: 1.5x

2. 高コントラスト有効化:
   設定 > アクセシビリティ > ハイコントラスト: ダーク

3. UIを近づける:
   設定 > 表示 > UIレイアウト > 距離: 1.5m
```

#### Q5: 音声コマンドが認識されない

**原因**:
- マイク権限がない
- 言語設定が違う
- 周囲がうるさい

**解決策**:
```
1. マイク権限許可:
   ブラウザ設定 > 権限 > マイク: 許可

2. 言語確認:
   設定 > 音声 > 言語: 日本語

3. 静かな環境で再試行

4. 音声コマンド再トレーニング:
   設定 > 音声 > トレーニング
```

#### Q6: 360°動画が再生されない

**原因**:
- フォーマット非対応
- ファイルサイズが大きすぎる
- ネットワークが遅い

**解決策**:
```
1. 対応フォーマット確認:
   - MP4 (H.264/H.265)
   - WebM (VP9)
   - 最大4K解像度

2. 品質設定を下げる:
   動画プレーヤー > 設定 > 品質: 1080p

3. 動画をプリロード:
   設定 > パフォーマンス > プリロード: オン

4. ネットワーク確認:
   Wi-Fi強度を確認
```

#### Q7: ジェスチャーマクロが動作しない

**原因**:
- 録画が完了していない
- ジェスチャーの順序が違う
- 感度設定が低い

**解決策**:
```
1. マクロを再録画:
   メニュー > ジェスチャー > 録画開始
   ゆっくり明確に実行
   録画停止

2. 感度を上げる:
   設定 > 入力 > ジェスチャー > 感度: 高

3. プリセットマクロを試す:
   メニュー > ジェスチャー > プリセット
```

#### Q8: UIが視野外にある

**原因**:
- レイアウト設定が極端
- アンカーモードが不適切
- カメラ位置がずれた

**解決策**:
```
1. レイアウトをリセット:
   設定 > 表示 > UIレイアウト > デフォルトに戻す

2. アンカーモード変更:
   設定 > 表示 > アンカーモード: Lazy-Follow

3. VR再起動:
   VRモード終了 → 再起動
```

## よくある質問（FAQ）

**Q: 無料で使えますか？**
A: はい、Qui Browser VRは完全無料のオープンソースソフトウェアです（MITライセンス）。

**Q: オフラインで使えますか？**
A: はい、PWA（Progressive Web App）としてインストールすれば、キャッシュされたページをオフラインで閲覧できます。

**Q: どのブラウザでも動作しますか？**
A: WebXR対応ブラウザ（Chrome、Edge、Firefox等）で動作します。Meta Quest内蔵ブラウザを推奨します。

**Q: コントローラーなしで使えますか？**
A: はい、ハンドトラッキング、視線コントロール、音声コマンドで完全にコントローラーレス操作可能です。

**Q: 日本語以外の言語は？**
A: 現在、日本語と英語をサポートしています。今後他言語も追加予定です。

**Q: カスタム拡張機能は使えますか？**
A: v2.2.0で拡張機能システムを実装予定です。現在は標準機能のみです。

**Q: データはどこに保存されますか？**
A: すべてローカル（ブラウザのlocalStorage）に保存されます。サーバーには送信されません。

**Q: マルチプレイヤーはできますか？**
A: v2.1.0で共有セッション機能を実装予定です。

**Q: 商用利用できますか？**
A: はい、MITライセンスのため商用利用可能です。

**Q: 貢献するには？**
A: GitHubリポジトリでプルリクエストを歓迎します！
